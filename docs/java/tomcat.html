# tomcat 下载

官网地址

# windwos 下启动tomcat

  1. 进入tomcat文件夹的bin目录找到startup.bat文件启动

  2. 进入/conf文件夹找到server.xml文件将port端口号更改，防止端口冲突

# 服务器部署tomcat

  • 下载相应jdk，直接安装，然后配置环境变量

  • 下载相应版本tomcat，安装tomcat 或直接复制tomcat文件夹

  • 调整tomcat/conf/文件夹下server.xml的端口号

  • 如启动时出现一闪而退，大概率是因为java—jdk路径未设置正确，需要更改tomcat/bin/setclasspath.bat || setclasspath.sh

  • 最后我特么重新粘贴一份tomcat文件就好使了！（估计是缺少文件了～）

# tomcat配置后台自动启动

windows server tomcat/bin目录下 启动 server.bat 如需开机启动，需要在服务中将服务配置为自动 linux server

# tomcat配置Https访问

  • 自行下载证书，然后解压证书，将证书处理为jks格式，网上自行搜索

  • 然后配置tomcat/conf/server.xml文件增加或解开注释

port 端口号 443是默认端口，8443在访问时需要在域名后追加端口号

keystoreFile 证书绝对路径

keystorePass 证书密码

<Connector port="8443" protocol="HTTP/1.1" SSLEnabled="true"
               maxThreads="150" scheme="https" secure="true"
               clientAuth="false" sslProtocol="TLS" 
               keystoreFile="C:\www\tomcat-ylfw-api\conf\domains.jks"
               keystorePass="******"  />

# war包在tomcat部署

将打包好的war包拷贝到tomcat/webapps目录下，启动tomcat，访问路径locallhost:port/projectname/, 如tomcat下只有一个项目可以将解压后的war包重命名为ROOT，这样访问路径默认为locallhost:port。也可以配置server.xml文件夹重定向地址。 找到host节点，配置如下内容


<Host appBase="webapps" autoDeploy="true" name="localhost" unpackWARs="true" xmlNamespaceAware="false" xmlValidation="false">
     <!-- path留空代表访问域名后面不需要带项目的名称 -->
     <Context path="" docBase="apollo-erp-web" reloadable="false" />
</Host>

# tomcat 配置“X-Frame-Options头”

服务器(tomcat)上所有项目生效

tomcat目录/conf/web.xml中的搜索httpHeaderSecurity配置（低版本不支持，需Tomcat 7.0.69以上），将其前面的注释去掉即可。

<filter>
      <filter-name>httpHeaderSecurity</filter-name>
      <filter-class>org.apache.catalina.filters.HttpHeaderSecurityFilter</filter-class>
      <async-supported>true</async-supported>
      <init-param>
        <param-name>antiClickJackingEnabled</param-name>
        <param-value>true</param-value>
      </init-param>
      <init-param>
        <param-name>antiClickJackingOption</param-name>
        <param-value>SAMEORIGIN</param-value>
      </init-param>
</filter>
